Clipboard

Clipboard was a project I undertook in CS350, a class that taught the fundamentals of Software Engineering.

In this class each group had four members with a role for each member.

My group's task was to create a project-tracking taskboard.

The roles in this class were Requirements, Design, Code, and Test, rotating roles each milestone.

For this project, I was the Test role for the first milestone, and rotated through the roles in the order I mentioned them.

There were 7 milestones in this project, each representing important steps in the Software Engineering process.

We kept track of our progress on github in .md documents, and you can click through each milestone below to see a detailed view

{% include './Documents_as_html/Milestone-1/Index.html' %}

During milestone 1 I was given the test role, which had the simple role of setting up a host for our project. After meeting with our team we decided on using Vercel for our hosting service, with django as the backend. We decided on these for a few reasons, including the cost of hosting, the ease of use, and that our team already had experience with these tools. Of course, this decision was not without its drawbacks as well, using a serverless hosting service would mean that we would need to set up a database hosting service as well.

In Milestone 2, I was the Requirements role, which had the pivotal role of creating a detailed list of the features and user stories we were going to implement in our project. This was a very important step in the Software Engineering process, as it defined the functionalities and cutoff points in our project. After speaking with the team and deciding on functionalities, I turned the functionalities into user stories and created a product backlog. I prioritized the user stories to create a Minimum Viable Product, alongside potential future enhancements post "release" at the end of the semester.

In Milestone 3, I was the Design role, which had the responsibility of finishing design for our database structure on our project. I built on the work of my team members in the previous milestones to create a detailed database structure that would support the features we had planned. This included creating ERDs using DBML to visualize the relationships between our tables, and creating a detailed design document that would guide the development of our project. I created a reverse hierarchical relational database structure, where the tasks would store their tasklist, tasklist their project, and projects the user. I also connected our project with MongoDB, created a .env to secure our database, and created create, read, and delete operations for our project model. In addition, I also used a team member's previous work on a logo to create a favicon for our site.

In Milestone 4, I was the Code role, which had the responsibility of implementing the features we had planned in Milestone 2. I worked with my team members to decide which features we would be able to implement this milestone, since it was clear that we would not be able to implement all of them in the time we had. With our focus on test-driven development, I used my teammate's previous work on tests to create a test.py file I could add to as I worked on the features. Using this work, I was able to implement the views, CRUD operations, and tests for Projects, Tasklists, and Tasks. I also fleshed out our sign up page, and added issues I found to our github, so that we could work on them in the future alongside keeping track our progress.

In the final milestone I was the Design role, and my assignment in this role was to look into the possible future of this project and create a devops report. I checked the scalability and cost of the three different services we were using, Vercel for hosting, MongoDB for Databases, and Checkly for live service testing and alerts. Throughout this milestone I checked our current usage on each of these platforms, versus the maximum allowable usage on the free tier, alongside the scaling costs of these services if we outscale the free tier.
